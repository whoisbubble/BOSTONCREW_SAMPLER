# BOSTONCREW SAMPLER

**BOSTONCREW SAMPLER** - десктопное приложение для управления сэмплами, слайдами, медиа-превью, сценическим окном и внешним host-подключением через WebSocket.

Приложение собрано на **Qt 6 / QML / C++**. Интерфейс сделан в тёмной crimson/burgundy-палитре, без стандартной рамки Windows, с кастомной верхней панелью, закруглённым окном и компактной сеткой управления.

## Основная идея

Приложение предназначено для живого управления шоу, квизом, презентацией или интерактивным событием:

- быстро запускать аудио-сэмплы;
- переключать слайды и медиа на сценическом экране;
- назначать до 8 быстрых слайд-слотов на главное окно;
- добавлять к каждому медиафайлу свой звуковой cue;
- выбирать конкретное медиа из текущего слайда через cue-окно;
- управлять видео: пауза, рестарт, удержание последнего кадра, повтор;
- подключаться к внешнему host-у по WebSocket и отправлять команды.

## Главное окно

Главное окно открывается как основной рабочий пульт.

Верхняя панель содержит:

- кнопку настройки host-подключения;
- кнопку remote-окна для управления видео;
- свернуть;
- развернуть / восстановить;
- закрыть приложение.

При закрытии приложение вызывает сохранение текущих данных.

Основная область разделена на три зоны:

- быстрые слоты слайдов;
- превью текущего или следующего медиа;
- область пользовательских сэмплов со строкой состояния.

## Быстрые слоты слайдов

В левом верхнем блоке главного окна находятся 8 видимых быстрых слотов.

Поведение слотов:

- левый клик по заполненному слоту открывает назначенный слайд на сцене;
- правый клик открывает выбор слайда для этого слота;
- если включён режим редактирования, клик по слоту тоже открывает назначение;
- активный слот подсвечивается;
- пустой слот показывает свой номер.

Назначение слота выполняется через окно **Quick slot**:

- есть общий поиск по названию и типу слайда;
- можно создать новый слайд;
- можно очистить слот;
- выбор строки назначает слайд на слот.

## Фиксированные сэмплы

Под быстрыми слайдами находятся фиксированные кнопки:

- Player / P1;
- Like / OK;
- Dislike / NO;
- Timer;
- Pause / Play всех активных аудио;
- Stop всех активных аудио.

Поведение:

- левый клик запускает соответствующий фиксированный сэмпл;
- правый клик запускает сэмпл и затем переключает медиа вперёд;
- средний клик по кнопке Timer открывает окно таймера;
- если включён режим редактирования, клик по фиксированному сэмплу заменяет его файл;
- кнопка Pause ставит на паузу или продолжает все активные аудио;
- кнопка Stop останавливает все активные аудио.

Фиксированные сэмплы хранятся отдельно от обычных сэмплов.

## Область превью

Правый верхний блок показывает текущее или следующее медиа.

Доступные элементы:

- кнопка Previous переключает на предыдущее медиа текущего слайда;
- кнопка Next переключает на следующее медиа;
- счётчик показывает позицию вида `1/5`;
- кнопка Cue открывает окно выбора медиа текущего слайда;
- переключатель `CUR` / `NEXT` меняет режим превью между текущим и следующим медиа.

Для видео превью запускается без звука и ждёт появления первого кадра.

## Окно Cue / Slide selection

Окно **BOSTONCREW SAMPLER / Cue** показывает медиа текущего активного слайда в виде сетки mini-HD ячеек.

Поведение ячеек:

- левый клик открывает выбранное медиа и запускает первый обычный сэмпл из области сэмплов;
- правый клик открывает выбранное медиа и запускает второй фиксированный сэмпл OK;
- клик колесом открывает только выбранное медиа без дополнительного звука;
- выбранные ячейки затемняются и получают зелёную отметку, чтобы было видно, какие медиа уже включались;
- если медиа много, окно можно прокручивать;
- красная полоса справа показывает позицию прокрутки.

Окно работает только когда открыт активный слайд.

## Сценическое окно

Сценическое окно отображает активное медиа для зрителя.

Поведение:

- если найден второй монитор, сцена открывается на нём и разворачивается под его геометрию;
- если второго монитора нет, сцена открывается на основном экране;
- изображения показываются в режиме сохранения пропорций;
- видео ждёт первый готовый кадр перед показом, чтобы не появлялся чёрный экран при переключении между форматами;
- после завершения видео остаётся на последнем кадре;
- повтор видео включается отдельно в менеджере слайдов для конкретного видео;
- закрытие сценического окна сбрасывает активный слайд.

## Remote-окно

Окно **BOSTONCREW SAMPLER / Remote** управляет текущим видео на сцене.

Доступные команды:

- Pause / Play - поставить активное видео на паузу или продолжить;
- Restart - вернуть видео в начало и запустить заново.

Кнопки активны только если сцена открыта и текущее медиа является видео.

Remote-окно не является WebSocket-пультом. Это локальное окно управления сценическим видео.

## Таймер

Окно **BOSTONCREW SAMPLER / Timer** открывается средним кликом по кнопке Timer в главном окне.

Функции:

- старт;
- пауза;
- сброс;
- отображение времени в формате `hh:mm:ss`.

Состояние таймера визуально подсвечивает кнопку Timer на главном окне.

## Обычные сэмплы

Нижняя область главного окна содержит обычные пользовательские сэмплы.

Поведение:

- левый клик запускает сэмпл;
- правый клик запускает сэмпл и переключает медиа вперёд;
- кнопка `+` добавляет новый аудиофайл;
- кнопка настроек включает или выключает режим редактирования;
- при включённом режиме редактирования клик по сэмплу открывает окно настройки;
- Shift + клик по первому сэмплу выбирает его для перестановки;
- Shift + клик по второму сэмплу перемещает выбранный сэмпл на новую позицию;
- активный сэмпл подсвечивается;
- маленькая жёлтая точка на сэмпле означает, что у него включён режим остановки других звуков.

Поддерживаемые аудиоформаты:

- `mp3`;
- `wav`;
- `m4a`;
- `ogg`;
- `flac`.

## Настройка сэмпла

Окно настройки сэмпла позволяет изменить:

- название;
- громкость;
- флаг `Stop other sounds`;
- цвет сэмпла;
- аудиофайл;
- удалить сэмпл.

Флаг `Stop other sounds` означает, что при запуске этого сэмпла приложение сначала остановит все другие активные аудио, а затем запустит выбранный сэмпл.

## Менеджер слайдов

Окно **BOSTONCREW SAMPLER / Slides** управляет библиотекой слайдов.

Возможности:

- общий поиск по названию, типу, именам медиа и cue-сэмплам;
- создание нового слайда;
- выбор слайда;
- переименование выбранного слайда;
- изменение типа слайда;
- добавление медиа;
- открытие папки слайда;
- удаление слайда;
- просмотр списка медиа внутри слайда.

При создании нового слайда приложение создаёт папку в `Content/NewSlide_N`.

Поддерживаемые медиаформаты при добавлении:

- изображения: `jpg`, `jpeg`, `png`, `bmp`;
- видео: `mp4`, `avi`, `mov`, `wmv`, `m4v`, `mkv`, `webm`.

## Медиа внутри слайда

Каждый слайд может содержать несколько медиафайлов.

Для каждого медиа доступны действия:

- переместить выше / левее в порядке воспроизведения;
- переместить ниже / правее;
- включить или выключить повтор для видео;
- добавить аудио cue к конкретному медиа;
- правый клик по кнопке audio удаляет cue у конкретного медиа;
- удалить медиа.

Если у медиа есть cue, при показе этого медиа автоматически запускается назначенный аудиофайл.

Повтор доступен только для видео. Если повтор выключен, видео после завершения остаётся на последнем кадре.

## Host и WebSocket

Host-подключение открывается кнопкой host в верхней панели.

Формат адреса:

```text
192.168.0.10:81
```

Приложение подключается как WebSocket-клиент по адресу:

```text
ws://192.168.0.10:81
```

Логика подключения:

1. Пользователь вводит host и нажимает `Connect`.
2. Backend открывает TCP-соединение через `QTcpSocket`.
3. После TCP-подключения отправляется WebSocket handshake.
4. Если host отвечает HTTP `101 Switching Protocols`, приложение считает WebSocket подключённым.
5. Адрес host сохраняется в `SaveData/host.txt`.
6. Индикатор подключения в строке состояния становится зелёным.

После успешного подключения приложение сразу отправляет текстовые WebSocket-сообщения:

```text
HOST
HSFALSE
```

Во время работы отправляются команды:

```text
RIGHT
WRONG
```

`RIGHT` отправляется при запуске фиксированного сэмпла OK.

`WRONG` отправляется при запуске фиксированного сэмпла NO.

Входящие WebSocket-сообщения:

```text
Игрок 1
Игрок 2
```

Если приложение получает одно из этих сообщений, оно запускает первый фиксированный сэмпл P1.

Технические особенности:

- используется обычный `ws://`, не `wss://`;
- сообщения отправляются только если соединение полностью установлено;
- если соединения нет, отправка silently игнорируется;
- клиентские WebSocket-фреймы маскируются;
- приложение ожидает от host-а стандартный WebSocket upgrade.

## Строка состояния

Внизу главного окна отображается:

- индикатор подключения к host;
- текущий статус приложения;
- режим `LIVE` или `EDIT`.

Примеры статусов:

- `Ready`;
- `Stage active`;
- `Connecting to host...`;
- `Host connected.`;
- `Host disconnected.`;
- сообщение об ошибке файла или подключения.

## Сохранение данных

Данные сохраняются автоматически после основных действий и при закрытии приложения.

Основные папки рядом с exe-файлом:

```text
SaveData/
Samples/
Content/
```

`SaveData/` хранит JSON-конфигурацию:

```text
samples.json
fixedsamples.json
slides.json
eightslides.json
host.txt
```

Для JSON-файлов также создаются `.bak` копии. Если основной JSON повреждён, приложение пробует прочитать backup.

`Samples/` хранит копии обычных и фиксированных аудиосэмплов.

`Content/` хранит папки слайдов, медиафайлы и cue-аудио, привязанные к медиа.

## Важные сценарии работы

### Подготовить сэмплы

1. Нажать `+` в области сэмплов.
2. Выбрать аудиофайл.
3. При необходимости включить режим настроек.
4. Открыть сэмпл и настроить название, громкость, цвет и `Stop other sounds`.

### Подготовить слайд

1. Открыть менеджер слайдов.
2. Нажать `+`.
3. Выбрать созданный слайд.
4. Добавить изображения или видео.
5. При необходимости добавить cue-звук к отдельным медиа.
6. Для видео включить repeat, если оно должно повторяться.

### Назначить слайд на главный экран

1. Правым кликом нажать на быстрый слот.
2. Найти нужный слайд через поиск.
3. Нажать на строку слайда.

### Запустить шоу

1. Левым кликом нажать на быстрый слот.
2. Сценическое окно откроется на втором мониторе или на основном экране.
3. Управлять медиа кнопками Previous / Next.
4. Открывать Cue-окно для выбора конкретного медиа.
5. Запускать сэмплы из нижней области.

### Подключить host

1. Нажать кнопку host в верхней панели.
2. Ввести адрес в формате `ip:port`.
3. Нажать `Connect`.
4. Дождаться статуса `Host connected.`.

## Сборка из исходников

Требования:

- Qt 6.10 или новее;
- CMake;
- компилятор C++17;
- Qt-модули `Quick`, `QuickControls2`, `Multimedia`, `Network`, `Widgets`.

Открыть проект можно через Qt Creator как CMake-проект.

Пример сборки из PowerShell:

```powershell
$env:PATH='C:\Qt\Tools\mingw1310_64\bin;C:\Qt\6.10.2\mingw_64\bin;' + $env:PATH
& 'C:\Qt\Tools\CMake_64\bin\cmake.exe' --build build-cmake -- -j1
```

Для deploy-сборки можно использовать скрипт:

```powershell
.\scripts\deploy_windows.ps1
```

## Структура проекта

Основные файлы:

- `main.cpp` - запуск приложения, регистрация backend в QML, настройка имени и иконки;
- `SamplerBackend.h/.cpp` - основная C++-логика: сэмплы, слайды, сохранение, WebSocket, сцена;
- `Main.qml` - главное окно;
- `DashboardView.qml` - раскладка главного экрана;
- `QuickSlotsPanel.qml` - быстрые слоты и фиксированные сэмплы;
- `StagePanel.qml` - превью текущего/следующего медиа;
- `SamplesConsole.qml` - нижняя область сэмплов;
- `SampleDialog.qml` - настройка сэмпла;
- `SlideManagerWindow.qml` - окно менеджера слайдов;
- `SlideLibrary.qml` - библиотека слайдов и медиа;
- `PreviewSelectWindow.qml` и `PreviewWorkspace.qml` - cue-выбор медиа;
- `StageWindow.qml` - сценическое окно;
- `RemoteWindow.qml` - локальное управление видео;
- `HostPopup.qml` - настройка host-подключения;
- `TimerWindow.qml` - таймер;
- `AppTheme.qml` - цвета, размеры и общая тема интерфейса;
- `assets/` - иконка приложения и SVG-иконки интерфейса.

## Ограничения и заметки

- Host-подключение работает только через `ws://`.
- Приложение не создаёт WebSocket-сервер, а подключается к внешнему host-у.
- Для корректного сценического режима рекомендуется подключить второй монитор до запуска шоу.
- Продолжительность аудио определяется через `ffprobe`, если рядом с exe есть `ffmpeg/ffprobe.exe`; без него длительность может отображаться как `00:00`.
- Видео-превью в менеджере слайдов показывается иконкой, а не thumbnail-кадром.
- Удаление слайда удаляет его папку из `Content/`, если она находится внутри рабочей директории приложения.
